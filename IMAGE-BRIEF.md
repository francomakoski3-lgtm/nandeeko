## 3. Nueva estrategia para FBA-002

FBA-002 no debe mostrarse como un solo render aislado.
La estrategia correcta es mostrarlo como una serie de desarrollo.

### Estructura recomendada

1. Una imagen general del sistema completo.
2. Una imagen hero por cada variante.
3. Detalles solo para la variante finalista o para las dos variantes finalistas.

### Por que esta estrategia es mejor

- evita comunicar una sola solucion cuando todavia hay cuatro caminos abiertos
- ordena mejor la conversacion con proveedores
- permite comparar variantes con consistencia visual real
- evita producir renders de detalle innecesarios antes de decidir la variante correcta

## 5. Criterios visuales para el disenador

### Direccion de arte

- estilo editorial y premium
- fondo calido y limpio
- props discretos
- misma luz para todas las variantes
- misma camara para que la comparacion sea honesta

### Lo que hay que mantener igual entre variantes FBA-002

- altura de camara
- distancia focal
- angulo del producto
- intensidad de luz
- fondo
- tipo de escritorio o superficie

Si eso cambia entre variantes, la comparacion visual pierde calidad.

### Lo que no debe aparecer

- texto dentro de la imagen
- flechas
- claims
- colores saturados
- props infantiles
- sombras duras
- encuadres extremos
